Terminologie si Structura

Doua Moduri de a Vedea un CNN

Figura 9.7 prezinta doua conventii de terminologie pentru descrierea CNN-urilor. In terminologia "complexa" (stanga), un singur layer CNN are mai multe "etape" interne: convolutie, detector (neliniaritate), pooling. In terminologia "simpla" (dreapta), fiecare etapa este considerata un layer separat. Cartea foloseste terminologia complexa.

In Practica: PyTorch/TensorFlow

Bibliotecile moderne folosesc terminologia simpla. Un "Conv2d" face doar convolutia, trebuie sa adaugi separat activarea (ReLU) si pooling-ul. Avantajul: flexibilitate maxima - poti folosi alte activari, poti sari pooling-ul, poti adauga BatchNorm intre etape, etc.

Simulare: Un Block CNN in PyTorch â–¶
import torch.nn as nn class CNNBlock(nn.Module): """Un 'layer complex' in terminologia cartii""" def __init__(self, in_ch, out_ch): super().__init__() # Etapa 1: Convolutie (transformare afina) self.conv = nn.Conv2d(in_ch, out_ch, kernel_size=3, padding=1) # Optional: Batch Normalization self.bn = nn.BatchNorm2d(out_ch) # Etapa 2: Detector (neliniaritate) self.relu = nn.ReLU() # Etapa 3: Pooling self.pool = nn.MaxPool2d(kernel_size=2, stride=2) def forward(self, x): x = self.conv(x) # Conv x = self.bn(x) # BatchNorm (optional) x = self.relu(x) # ReLU x = self.pool(x) # MaxPool return x # Utilizare block = CNNBlock(3, 64) # RGB -> 64 filtre
