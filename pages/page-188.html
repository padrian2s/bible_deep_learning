6.1 XOR: Transformarea Spatiului de Caracteristici

Figura 6.1: Rezolvarea XOR prin Invatarea unei Reprezentari

Aceasta figura ilustreaza cum reteaua neuronala rezolva problema XOR. In spatiul original x, un model liniar nu poate separa clasele deoarece punctele cu output 1 (la (0,1) si (1,0)) si cele cu output 0 (la (0,0) si (1,1)) nu sunt liniar separabile. In spatiul transformat h, invatat de retea, problema devine liniar separabila.

Colapsarea Punctelor in Spatiul Feature

In solutia noastra, cele doua puncte care trebuie sa outputeze 1, x = [1,0]·µÄ si x = [0,1]·µÄ, au fost colapsate intr-un singur punct in spatiul feature, h = [1,0]·µÄ. Caracteristicile neliniare au mapat ambele puncte la aceeasi reprezentare. Modelul liniar poate acum descrie functia ca fiind crescatoare in h‚ÇÅ si descrescatoare in h‚ÇÇ.

Simulare: Maparea x ‚Üí h ‚ñ∂
import numpy as np # Transformarea invatata de retea def transform_to_h(x, W, c): """h = ReLU(xW + c)""" return np.maximum(0, x @ W + c) # Parametrii invatati W = np.array([[1, 1], [1, 1]]) c = np.array([0, -1]) # Toate punctele XOR X = np.array([[0, 0], [0, 1], [1, 0], [1, 1]]) # Transformare H = transform_to_h(X, W, c) print("x ‚Üí h:") print("[0,0] ‚Üí", H[0]) # [0, 0] print("[0,1] ‚Üí", H[1]) # [1, 0] ‚Üê COLAPSATE print("[1,0] ‚Üí", H[2]) # [1, 0] ‚Üê LA ACELASI PUNCT! print("[1,1] ‚Üí", H[3]) # [2, 1] # Acum in spatiul h, clasele sunt separabile!

Intuitive: De ce functioneaza ‚ñ∂
Ideea cheie: Reteaua invata o reprezentare in care punctele similare (din perspectiva task-ului) sunt apropiate, iar punctele diferite sunt departate. XOR: (0,1) si (1,0) sunt "similare" (ambele ‚Üí 1), deci reteaua le mapeaza aproape in spatiul h.

In aplicatii reale, reprezentarile invatate ajuta si la generalizare - nu doar la potrivirea datelor de antrenare. Imaginile de pisici vor fi mapate aproape in spatiul feature, chiar daca pixelii sunt foarte diferiti.
