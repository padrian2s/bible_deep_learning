Ce face o reprezentare "buna"?

In contextul ML, o reprezentare buna este una care face task-ul de invatare ulterior mai usor. Retelele feedforward supervizate pot fi vazute ca facand representation learning: straturile ascunse invata o reprezentare astfel incat ultimul strat (clasificator liniar/softmax) sa poata rezolva task-ul. Clasele care nu erau liniar separabile in input devin liniar separabile in reprezentarea din ultimul hidden layer!

Tradeoff-ul Reprezentarilor

Majoritatea problemelor de representation learning implica un tradeoff intre a pastra cat mai multa informatie despre input si a obtine proprietati "frumoase" (precum independenta statistica). Representation learning este interesant pentru ca permite invatare semi-supervizata: avem multe date nelabeled si putine labeled. Antrenarea supervizata doar pe datele labelled duce la overfitting; invatam reprezentari din datele unlabeled, apoi le folosim pentru task-ul supervizat.

Semi-Supervised Learning â–¶
De ce Semi-Supervised?
  • Date unlabeled: Foarte multe, usor de obtinut
  • Date labeled: Putine, scumpe (necesita experti)
  • Solutia: Invata reprezentari din unlabeled, apoi fine-tune pe labeled

Oamenii si animalele invata din foarte putine exemple labelled - probabil folosesc reprezentari invatate nesupervizat.
