Simulare: Implementare in Python â–¶
import numpy as np def conv2d_valid(image, kernel): """Convolutie 2D 'valid' - fara padding""" ih, iw = image.shape kh, kw = kernel.shape oh, ow = ih - kh + 1, iw - kw + 1 output = np.zeros((oh, ow)) for i in range(oh): for j in range(ow): # Extragem regiunea din imagine region = image[i:i+kh, j:j+kw] # Inmultim element-wise si sumam output[i, j] = np.sum(region * kernel) return output # Exemplu din figura image = np.array([ [1, 2, 3, 4], # a, b, c, d [5, 6, 7, 8], # e, f, g, h [9, 10, 11, 12] # i, j, k, l ]) kernel = np.array([ [1, 2], # w, x [3, 4] # y, z ]) output = conv2d_valid(image, kernel) print("Output:\n", output) # [[ 44 54 64] # [ 84 94 104]]

Valid Convolution - Restrictia Kernelului

In "valid" convolution, outputul este calculat doar in pozitiile unde kernelul se incadreaza complet in imagine. Aceasta inseamna ca outputul este MAI MIC decat inputul: pentru input HxW si kernel KhxKw, outputul are dimensiunea (H-Kh+1) x (W-Kw+1). In exemplul nostru: 3x4 input, 2x2 kernel â†’ 2x3 output.

Tipuri de Padding â–¶
VALID
Fara padding

Output mai mic. Folosit cand nu vrem informatii "de margine".

SAME
Padding cu zerouri

Output = Input size. Cel mai comun in CNN-uri moderne.

FULL
Padding maxim

Output mai mare. Fiecare pixel participa in toate pozitiile posibile ale kernelului.

Simulare: Comparatie Padding â–¶
import torch import torch.nn.functional as F x = torch.randn(1, 1, 5, 5) # Input 5x5 kernel = torch.randn(1, 1, 3, 3) # Kernel 3x3 # Valid (padding=0) out_valid = F.conv2d(x, kernel, padding=0) print("Valid:", out_valid.shape) # [1,1,3,3] # Same (padding=1 pentru kernel 3x3) out_same = F.conv2d(x, kernel, padding=1) print("Same:", out_same.shape) # [1,1,5,5] # Full (padding=2 pentru kernel 3x3) out_full = F.conv2d(x, kernel, padding=2) print("Full:", out_full.shape) # [1,1,7,7]
