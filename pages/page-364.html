9.5 Zero-Padding

Valid, Same si Full Convolution

Problema: Dimensiunea Output-ului Scade

Fara padding, fiecare layer convolutional reduce dimensiunea spatiala cu (kernel_size - 1). Pentru o imagine 32x32 cu kernel 3x3, output-ul e 30x30. Dupa cateva layere, dimensiunea devine prea mica. Zero-padding rezolva aceasta problema adaugand zerouri in jurul inputului.

Trade-off-uri ale Padding-ului

Valid: Pixelii de margine "vad" mai putine date, comportament mai regulat, dar dimensiunea scade. Same: Cel mai popular - mentine dimensiunea, dar pixelii de margine sunt "subreprezentati" (vad mai multe zerouri). Full: Fiecare pixel contribuie egal la toate pozitiile posibile, dar outputul e mai mare si margini "artificiale".

Simulare: Padding in PyTorch â–¶
import torch import torch.nn as nn x = torch.randn(1, 3, 32, 32) # Input 32x32 # VALID - no padding conv_valid = nn.Conv2d(3, 64, kernel_size=5, padding=0) print("Valid:", conv_valid(x).shape) # [1, 64, 28, 28] # SAME - padding = kernel_size // 2 conv_same = nn.Conv2d(3, 64, kernel_size=5, padding=2) print("Same:", conv_same(x).shape) # [1, 64, 32, 32] # SAME (automatic in PyTorch) conv_same_auto = nn.Conv2d(3, 64, kernel_size=5, padding='same') print("Same auto:", conv_same_auto(x).shape) # [1, 64, 32, 32] # FULL - padding = kernel_size - 1 conv_full = nn.Conv2d(3, 64, kernel_size=5, padding=4) print("Full:", conv_full(x).shape) # [1, 64, 36, 36] # Formula generala: # output_size = (input_size + 2*padding - kernel_size) / stride + 1

Regula Practica â–¶
In majoritatea cazurilor, foloseste padding='same' (sau padding=k//2 pentru kernel de dimensiune k).

Aceasta permite stivuirea oricarui numar de convolutii fara grija pentru dimensiuni. Downsampling-ul se face explicit prin stride sau pooling, nu prin "pierdere accidentala" de dimensiune.
