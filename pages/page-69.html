Informatica vs Machine Learning: Certitudine vs Incertitudine

Multe ramuri ale informaticii lucreaza cu entitati deterministe si certe. Un programator poate presupune ca CPU-ul va executa fiecare instructiune impecabil. Dar Machine Learning trebuie MEREU sa lucreze cu cantitati incerte si uneori stocastice (non-deterministe). Incertitudinea poate veni din mai multe surse, iar teoria probabilitatilor ofera argumente convingatoare pentru cuantificarea acestei incertitudini inca din anii 1980 (Pearl, 1988).

Sursa 1: Stocasticitate Inerenta

Unele sisteme sunt intrinsec aleatorii. De exemplu, mecanica cuantica descrie dinamica particulelor subatomice ca fiind probabilista. Sau, in scenarii teoretice precum jocurile de carti, presupunem ca acestea sunt amestecate aleatoriu. Chiar daca lumea reala ar fi determinista la nivel fundamental, existenta aleatoriului este utila conceptual pentru modelarea sistemelor complexe.

üéÆ
Simulare: Joc de carti aleatoriu ‚ñ∂
import numpy as np # Simulare: Probabilitatea de a primi un As def simulate_card_draws(n_simulations=10000): deck = list(range(52)) # 52 carti, 4 Asi (0-3) aces_count = 0 for _ in range(n_simulations): np.random.shuffle(deck) first_card = deck[0] if first_card < 4: # Este un As aces_count += 1 empirical_prob = aces_count / n_simulations theoretical_prob = 4 / 52 print(f"P(As) empiric: {empirical_prob:.4f}") print(f"P(As) teoretic: {theoretical_prob:.4f}") # Output: ~0.0769 pentru ambele simulate_card_draws()

Sursa 2: Observabilitate Incompleta

Chiar si sisteme deterministe pot parea stocastice cand nu putem observa toate variabilele. Exemplul clasic este Problema Monty Hall: un concurent trebuie sa aleaga intre trei usi, doua ducand la o capra si una la o masina. Dupa ce alegi, gazda deschide o usa cu o capra. Ar trebui sa schimbi alegerea? Din perspectiva ta, rezultatul e incert - desi sistemul e complet determinist!

üéÆ
Simulare: Problema Monty Hall ‚ñ∂
import numpy as np def monty_hall_simulation(n_games=10000): wins_stay = 0 wins_switch = 0 for _ in range(n_games): # Masina e in spatele unei usi aleatoare (0, 1, sau 2) car_door = np.random.randint(3) # Concurentul alege o usa player_choice = np.random.randint(3) # Strategia 1: Ramai la alegerea initiala if player_choice == car_door: wins_stay += 1 # Strategia 2: Schimba mereu (dupa ce gazda arata o capra) # Daca schimbi, castigi DOAR daca initial ai ales gresit if player_choice != car_door: wins_switch += 1 print(f"Castig ramanand: {wins_stay/n_games:.2%}") print(f"Castig schimband: {wins_switch/n_games:.2%}") # Output: ~33% vs ~67% - schimbarea e MEREU mai buna! monty_hall_simulation()
Intuitie: Initial ai 1/3 sanse sa fi ales corect. Dupa ce gazda arata o capra, probabilitatea de 2/3 sa fi gresit se "concentreaza" pe usa ramasa!

Sursa 3: Modelare Incompleta

Cand folosim un model care trebuie sa ignore parte din informatia observata, informatia ignorata rezulta in incertitudine in predictii. De exemplu, un robot care poate observa locatia exacta a fiecarui obiect, dar discretizeaza spatiul in celule - obiectele pot fi oriunde in celula respectiva, introducand incertitudine.

üéÆ
Simulare: Discretizare si incertitudine ‚ñ∂
import numpy as np # Pozitie exacta vs discretizata exact_position = 3.7 # metri cell_size = 1.0 # metru per celula # Discretizare cell_index = int(exact_position // cell_size) # = 3 print(f"Obiect in celula {cell_index}") # Incertitudine introdusa # Obiectul poate fi oriunde intre 3.0 si 4.0 metri uncertainty_range = (cell_index * cell_size, (cell_index + 1) * cell_size) print(f"Pozitie posibila: {uncertainty_range}") # (3.0, 4.0) - am pierdut precizia de 0.7m! # In ML: feature discretization creeaza aceeasi problema # Ex: varsta 25 -> categoria "20-30" -> pierdere informatie
