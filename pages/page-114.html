5.1 Learning Algorithms - Definitia lui Mitchell

Un algoritm de machine learning este un algoritm care poate invata din date. Dar ce inseamna "a invata"? Mitchell (1997) ofera definitia: "Un program invata din experienta E in raport cu o clasa de task-uri T si o masura de performanta P, daca performanta sa la task-urile din T, masurata prin P, se imbunatateste cu experienta E." Aceasta carte ofera descrieri intuitive ale task-urilor, masurilor de performanta si experientelor care pot fi folosite pentru a construi algoritmi ML.

# Exemplu concret: Clasificare de imagini # T (Task): Clasificarea imaginilor in categorii task = "Clasificare: imagine -> eticheta (pisica/caine)" # P (Performance): Acuratetea pe date de test def performance(predictions, true_labels): correct = sum(p == t for p, t in zip(predictions, true_labels)) return correct / len(true_labels) # E (Experience): Dataset de antrenament experience = { 'images': ['img1.jpg', 'img2.jpg', ...], 'labels': ['pisica', 'caine', ...] } # INVATARE = imbunatatirea lui P prin E from sklearn.neural_network import MLPClassifier model = MLPClassifier() # Inainte de invatare accuracy_before = 0.5 # Random guessing # Dupa invatare din experienta model.fit(experience['images'], experience['labels']) accuracy_after = 0.95 # Performanta imbunatatita! print(f"Performanta s-a imbunatatit: {accuracy_before} -> {accuracy_after}") print("=> Modelul a INVATAT!")

5.1.1 The Task, T

Machine learning ne permite sa rezolvam task-uri prea dificile pentru a fi rezolvate cu programe fixe scrise de oameni. Din punct de vedere stiintific si filosofic, ML este interesant pentru ca dezvoltarea intelegerii noastre implica dezvoltarea principiilor care stau la baza inteligentei. In aceasta definitie, procesul de invatare nu este task-ul in sine. Task-ul este abilitatea pe care vrem sa o dobandim. De exemplu, daca vrem ca un robot sa mearga, mersul este task-ul - invatarea este mijlocul de a obtine aceasta abilitate.
