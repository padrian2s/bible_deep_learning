Generalizarea prin Atribute Partajate

Ce distinge reprezentarile distribuite de cele simbolice este ca generalizarea apare din atribute partajate. Ca simboluri pure, "cat" si "dog" sunt la aceeasi distanta ca orice alte doua simboluri. Dar cu o reprezentare distribuita, pot avea atribute comune: "has_fur", "number_of_legs=4". Ceea ce invatam despre pisici poate generaliza la caini prin aceste atribute partajate!

# Reprezentari simbolice vs distribuite pentru animale # ONE-HOT: nu partajeaza nimic cat_onehot = [1, 0, 0, 0] dog_onehot = [0, 1, 0, 0] car_onehot = [0, 0, 1, 0] # Toate perechi sunt la distanta sqrt(2) - nu e generalizare! # DISTRIBUTED: atribute partajate # [fur, 4_legs, pet, vehicle] cat_dist = [1, 1, 1, 0] # blana, 4 picioare, animal de companie dog_dist = [1, 1, 1, 0] # IDENTIC cu cat pe aceste atribute! car_dist = [0, 0, 0, 1] # total diferit # Generalizare: ce invatam pentru cat (are blana -> caldura) # se aplica automat la dog (are si el blana!) print("Cat-Dog distance:", sum((a-b)**2 for a,b in zip(cat_dist, dog_dist))) # 0! print("Cat-Car distance:", sum((a-b)**2 for a,b in zip(cat_dist, car_dist))) # 4

Spatiul de Similaritate

Reprezentarile distribuite induc un spatiu de similaritate bogat in care concepte semantic apropiate sunt aproape in distanta. Aceasta proprietate e absenta din reprezentarile simbolice. Word embeddings (Word2Vec, GloVe) functioneaza exact pe acest principiu - cuvinte similare au vectori apropiati, permitand generalizarea la cuvinte nevazute in anumite contexte.
