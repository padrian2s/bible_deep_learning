15.4 Distributed Representation

Reprezentarile distribuite - compuse din multe elemente care pot fi setate independent - sunt unul dintre cele mai importante instrumente pentru representation learning. Cu n features si k valori posibile, putem descrie k^n configuratii diferite! Compara cu reprezentarile simbolice (one-hot) unde n simboluri pot descrie doar n configuratii. Reprezentarile distribuite sunt exponential mai expresive.

# Reprezentari simbolice vs distribuite # ONE-HOT (simbolic): n simboluri = n regiuni n_symbols = 1000 # vocabular de 1000 cuvinte regions_symbolic = n_symbols # poate distinge 1000 concepte # DISTRIBUTED: n features binare = 2^n regiuni n_features = 10 # doar 10 features! regions_distributed = 2 ** n_features # 1024 configuratii! print(f"Simbolic (1000 dim one-hot): {regions_symbolic} regiuni") print(f"Distribuit (10 features binare): {regions_distributed} regiuni") # Cu 20 features binare: print(f"Distribuit (20 features): {2**20:,} regiuni!") # 1,048,576! # Concluzie: reprezentarile distribuite sunt EXPONENTIAL # mai expresive decat cele simbolice!

Exemple de Reprezentari Non-Distribuite

Algoritmi care folosesc reprezentari non-distribuite: k-means (un cluster per input), k-nearest neighbors (templates discrete), decision trees (o frunza per regiune), Gaussian mixtures, n-gram language models. Aceste metode imparte spatiul in regiuni separate cu parametri independenti per regiune - nu partajeaza informatie intre regiuni vecine!

Reprezentari Distribuite in NLP

Revolutia in NLP a fost trecerea de la one-hot la distributed:

One-hot (vechi)

"cat" = [0,0,1,0,0,...,0]

50K dim, sparse, fara semantica

Distributed (Word2Vec+)

"cat" = [0.2, -0.5, 0.8, ...]

300 dim, dense, capteaza semantica

Transformers: fiecare token = 768-4096 features distribuite!
