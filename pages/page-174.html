Figura 5.10: Nearest Neighbor Regions

Ilustratie a modului in care nearest neighbor imparte spatiul de input in regiuni. Un exemplu (reprezentat de un cerc) in fiecare regiune defineste region boundary (reprezentat prin linii). Valoarea y asociata cu fiecare exemplu defineste care ar trebui sa fie output-ul pentru toate punctele din regiunea corespunzatoare. Regiunile definite de nearest neighbor matching formeaza un pattern geometric numit Voronoi diagram.

Proprietati Voronoi
Definitie:

Regiunea Voronoi pentru punctul p = toate punctele mai aproape de p decat de orice alt punct

Boundary:

Linii drepte (perpendicular bisector intre puncte vecine)

Numar regiuni:

Exact n regiuni pentru n puncte

Simulare: Voronoi Regions â–¶
import numpy as np from sklearn.neighbors import KNeighborsClassifier np.random.seed(42) # 3 puncte de training X_train = np.array([[1, 2], [3, 1], [2, 3]]) y_train = np.array([0, 1, 2]) # fiecare punct = regiune separata # 1-NN classifier knn = KNeighborsClassifier(n_neighbors=1) knn.fit(X_train, y_train) # Grid de test x_range = np.linspace(0, 4, 10) y_range = np.linspace(0, 4, 10) print("Voronoi regions (regiune pentru fiecare punct):") print() for y in reversed(y_range): row = "" for x in x_range: region = knn.predict([[x, y]])[0] row += str(region) + " " print(row) print() print("Legend: 0, 1, 2 = regions defined by") print(" the 3 training points") print() print("=> Fiecare exemplu controleaza O SINGURA regiune")

Limitarea Fundamentala

Numarul de regiuni contigue (Voronoi cells) nu poate creste mai repede decat numarul de exemple de training. In timp ce aceasta figura ilustreaza comportamentul nearest neighbor specific, alte algoritmi ML care se bazeaza exclusiv pe local smoothness prior pentru generalizare exhiba comportamente similare: fiecare exemplu de training informeaza learner-ul doar despre cum sa generalizeze in vecinatatea imediata a acelui exemplu.
