Convolutii 1D, 2D si 3D

Tabelul 9.1 prezinta exemple de date pentru fiecare dimensionalitate. Convolutia 1D e pentru serii temporale (audio, text), 2D pentru imagini, si 3D pentru volume (CT scans, video). Fiecare poate avea un singur canal sau multiple canale.

Echivarianta in Fiecare Dimensiune

Convolutia 1D pe audio ofera echivarianta la translatii in timp - un sunet recunoscut la t=0 va fi recunoscut si la t=5s. Convolutia 2D pe spectrograme ofera echivarianta in timp SI frecventa - aceeasi melodie cantata la o octava diferita va produce aceeasi reprezentare (dar decalata pe axa frecventei).

Simulare: Conv1D pentru Audio â–¶
import torch import torch.nn as nn # Audio: 1 canal (mono), 16000 samples (1 sec @ 16kHz) audio = torch.randn(8, 1, 16000) # batch=8 # Conv1D pentru extragere features conv1d = nn.Sequential( nn.Conv1d(1, 64, kernel_size=80, stride=4), # ~5ms window nn.ReLU(), nn.Conv1d(64, 128, kernel_size=3), nn.ReLU(), nn.AdaptiveAvgPool1d(1) # Global pooling ) features = conv1d(audio) print("Audio features:", features.shape) # [8, 128, 1] # Alternativ: WaveNet, Wav2Vec folosesc convolutii dilatate # pentru receptive field foarte mare (secunde de audio)

Cand sa folosesti Conv1D vs Conv2D? â–¶
Conv1D pentru:

Serii temporale, text (caractere/embeddings), raw audio waveforms

Conv2D pentru:

Imagini, spectrograme, heatmaps, orice date cu structura grila 2D

Conv3D pentru:

Video, imagini medicale volumetrice (CT, MRI), point clouds voxelizate
