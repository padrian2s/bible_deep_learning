Problema Clustering-ului: Ill-Posed

O dificultate legata de clustering este ca problema este inherent ill-posed - nu exista un singur criteriu care masoara cat de bine corespondent clustering-ul datelor cu lumea reala. Putem masura proprietati precum distanta medie Euclidiana de la centroizi, dar nu stim daca asignarile clusterelor corespund proprietatilor reale. Pot exista multe clustering-uri diferite care corespund bine unor proprietati ale lumii reale. De exemplu, pentru imagini cu masini rosii, masini gri, camioane rosii si camioane gri, un algoritm poate gasi clustere "masini vs camioane", altul "rosii vs gri", si altul patru clustere!

✨
Vizualizare: Multiple Valid Clusterings ▶
Exemplu: Masini si Camioane
Clustering A (k=2):

Cluster 1: Masini | Cluster 2: Camioane

Clustering B (k=2):

Cluster 1: Rosii | Cluster 2: Gri

Clustering C (k=4):

Masini rosii | Masini gri | Camioane rosii | Camioane gri

Toate sunt "corecte" - depinde de ce proprietate ne intereseaza!

Distributed vs One-Hot Representations

Aceste probleme ilustreaza de ce preferam uneori o distributed representation in locul unei reprezentari one-hot. O reprezentare distribuita ar putea avea doua atribute pentru fiecare vehicul - unul pentru culoare si unul pentru masina-vs-camion. Nu este clar cum algoritmul de invatare stie ce atribute ne intereseaza (culoare vs producator?), dar avand multe atribute reduce presiunea de a ghici care e singurul atribut important si ne permite sa masuram similaritatea intre obiecte comparand mai multe atribute in loc de un singur cluster.

✨
Vizualizare: One-Hot vs Distributed ▶
One-Hot

[1, 0, 0, 0] = masina rosie

[0, 1, 0, 0] = masina gri

Similaritate: doar 0 sau 1

Distributed

[1, 0, 1, 0] = masina, rosie

[1, 0, 0, 1] = masina, gri

Similaritate: partial matching
